# Data Model

## patent.yaml — Source of Truth

All structured data lives in `patent.yaml`:

```yaml
metadata:
  title: "..."
  applicant: "..."
  inventors: [{name: "...", address: "..."}]
  filing_date: ""
  target_jurisdictions: [EP, US]
  related_applications: []
  government_funded: false

reference_numerals:
  - id: reactor-vessel          # stable slug
    label: "reactor vessel"     # text for EP claim insertion
    number: 100                 # current number (mutable)
    prev_numbers: []            # history
    introduced_in: reactor-overview  # figure slug

figures:
  - id: reactor-overview        # stable slug
    title: "..."
    description: "..."
    numerals_shown: [reactor-vessel, inlet-port]

claims:
  - id: mof-synthesis-method    # stable slug
    type: independent
    category: method
    depends_on: null             # slug of parent, or null
    body:
      preamble: "A method for ..."
      transitional: "comprising"
      elements:
        - text: "providing a reactor vessel"
          numerals: [reactor-vessel]
          spec_support: "sections/detailed-description.tex:10-15"
    reference_numerals_used: [reactor-vessel]

prior_art:
  - id: smith2023-mof-reactor
    citation: "..."
    doi: "..."
    relevance: "..."
    distinguishing_features: ["..."]
    claims_affected: [mof-synthesis-method]
    tome_key: smith2023

ids_submissions:
  - date: "2025-03-14"
    refs: [smith2023-mof-reactor]
    status: filed

glossary:
  - term: "reactor vessel"
    numeral: reactor-vessel
    aliases_rejected: ["reaction chamber", "reactor"]
```

## What's Authoritative vs Generated

| Data | Location | Edited by |
|------|----------|-----------|
| Claims, numerals, figures registry, glossary, prior art, metadata | patent.yaml | MCP tools |
| Background, detailed description, field, abstract | sections/*.tex | Windsurf text editing |
| Claims section, drawings description, numeral macros | sections/*.tex | GENERATED by export tools |

## Stable IDs

Every entity uses a kebab-case slug as its primary key. Presentation numbers (claim 1, FIG. 1, numeral 100) are computed or mutable — never used as identity.

## Dual Addressing

Tools accept slug OR current number for claims, figures, and numerals. For numerals, previous numbers are also checked.
